# 📄 File Wiki: index.html & demo.html

## Overview
`index.html` (and its live browser demonstration counterpart `demo.html`) forms the primary user interface structure for Tomb OS. It defines the GNOME-inspired glassmorphism desktop environment, activities bar, system tray indicators, window containers, installation setup modal, and mandatory administrative lock screen.

## Key Components & Architecture

### 1. Mandatory Installation Setup Modal (`#setup-modal`)
- **Purpose**: Prompts the user upon first boot to define a master administrative passphrase (`sec-admin`).
- **Security Interlocks**: Encrypts the local hardware TPM 2.0 enclave and binds administrative session privileges. Supports YubiKey / Titan NFC hardware 2FA attestation.
- **Legal Notices**: Embeds the mandatory verbatim legal disclaimer and Good Faith & World-Focused Action Charter.

### 2. Mandatory Administrative Lock Screen (`#session-lock-screen`)
- **Purpose**: Enforces zero-trust re-authentication whenever a session locks or user logs in.
- **Biometric / NFC Key Integration**: Allows quick-unlock using WebAuthn NFC security keys.

### 3. GNOME Desktop Top Bar (`#top-bar`)
- **Activities Button**: Triggers the Control Center launcher modal (`openWindow('controlcenter')`).
- **Live System Indicators**: Displays live SD Card Boot mode indicator, live clock, security shield status badge, and quick settings dropdown trigger.
- **AI Task Recorder Action Badge**: Allows instant toggle of the autonomous AI task recording daemon.

### 4. Application Windows Container (`#windows-container`)
- Dynamic DOM root where floating draggable windows generated by `app.js` (`openWindow()`) are injected and rendered.
